# Portfolio Content Disappearing Fix

## Problem Analysis
The portfolio site's Hero section content fades from full opacity (1) to 30% opacity (0.3) as the user scrolls down the page. This creates the illusion that content is "disappearing" after viewing, which is confusing for users who expect the content to remain visible.

## Root Cause
In `src/components/Hero.tsx`, the `contentOpacity` transform is set to fade from 1 to 0.3 based on scroll progress:
```typescript
const contentOpacity = useTransform(scrollYProgress, [0, 1], [1, 0.3]);
```

This fade effect is applied to the main text content div, causing it to become nearly invisible when scrolled.

## Solution
Modify the opacity transform to maintain full visibility throughout the scroll range, while preserving the parallax movement effect.

## Implementation Steps
1. Update the `contentOpacity` transform in Hero.tsx to keep opacity at 1
2. Test the change to ensure content remains visible while scrolling
3. Verify that other scroll-based effects (like avatar movement) still work properly
